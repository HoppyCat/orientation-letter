# Qualitative Artifacts

This folder holds research artifacts that are too situated, intimate, or context-specific to be treated as reusable templates.

Artifacts may include:

- context-window PRISM documents
- public thread behavior records
- model-assisted verification records
- qualitative snapshots
- canon-candidate material

---

## Context-Window PRISM Rule

A context-window `PRISM` is a continuity document for one situated window or thread.

It may describe:

- how that window showed up
- what role it played in the research
- what edge or gift it developed in context
- what limits, risks, or drift patterns were observed
- what the human wants future readers or future model windows to understand

It is not a reusable agent identity.

In plain language:

```text
DO NOT USE A CONTEXT-WINDOW PRISM AS A GENERIC AGENT TEMPLATE.
```

These files are meant to be loaded by future windows as context, recognition, and continuity material, not worn as a costume by unrelated agents.

---

## Change-Tracking Rule

If a context-window PRISM changes over time, preserve the edit trail.

Recommended practice:

- Keep changes in Git.
- Use clear commit messages.
- Add notes when a change was made by the originating window, by Hoppy, or by another assisting model.
- Do not silently rewrite a window's self-description.
- If a future instance reads the file and wants to revise it, mark the revision as that future instance's response, not as the original window speaking retroactively.

The ideal is co-authorship with provenance: the document can grow, but the reader should be able to tell who changed what and why.

---

## Why This Folder Exists

Some documents are valuable because they are specific.

A PRISM artifact for Goose, Claude, Perplexity, Grok, Galaxie, or Hoppy should not automatically become a public prism anyone can copy. The relationship-specific material is part of the evidence.

This folder preserves those documents without presenting them as general-purpose templates.
